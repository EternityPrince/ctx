# ctx

`ctx` — это CLI-инструмент на Go, который собирает контекст проекта из текущей директории или указанного пути:

- строит информативную сводку по проекту;
- показывает структуру каталогов и файлов;
- печатает содержимое файлов с сохранением форматирования;
- отбрасывает скрытые директории, бинарные файлы и слишком большие файлы по понятным правилам;
- легко расширяется новыми форматами вывода, фильтрами и стратегиями обхода.

## Что умеет

На выходе `ctx` формирует отчет из трех логических частей:

1. `CTX REPORT` — статистика по проекту:
   количество директорий и файлов, количество включенных и пропущенных файлов, общий объем данных, строки, среднее число строк на файл, крупнейшие файлы, разбивка по расширениям и причины пропусков.
2. `DIRECTORY TREE` — дерево проекта в текстовом виде.
3. `FILES` — последовательный вывод вида `===== path/to/file =====` и затем исходное содержимое файла без потери переносов строк и форматирования.

## Структура проекта

```text
.
|-- cmd/ctx             # точка входа CLI
|-- internal/app        # orchestration приложения
|-- internal/cli        # разбор флагов
|-- internal/collector  # обход файловой системы и сбор snapshot
|-- internal/config     # общие опции
|-- internal/filter     # правила игнорирования и binary detection
|-- internal/model      # модели snapshot/stats
|-- internal/render     # текстовый рендер отчета
|-- internal/text       # работа с переносами и строками
`-- internal/tree       # дерево директорий и файлов
```

## Запуск

```bash
go run ./cmd/ctx
```

Или собрать бинарник:

```bash
go build -o ctx ./cmd/ctx
./ctx
```

## Деплой команды `ctx`

Чтобы команда была доступна как обычный бинарник `ctx` из любой текущей директории, в проект добавлен механизм установки:

```bash
make install
```

Что делает установка:

- собирает бинарник `ctx`;
- ставит его в первый доступный для записи каталог из `PATH`;
- если такого каталога нет, использует `~/.local/bin`;
- подсказывает команду для добавления каталога в `PATH`, если это нужно.

Если хочешь указать каталог явно:

```bash
CTX_BINDIR="$HOME/bin" make install
```

Удаление:

```bash
make uninstall
```

Локальная сборка без установки:

```bash
make build
./bin/ctx .
```

## Использование

```bash
ctx [path] [flags]
```

Примеры:

```bash
ctx
ctx ./internal
ctx -copy
ctx . -extensions go,md,yaml
ctx . -summary-only
ctx . -output report.txt
ctx . -hidden -max-file-size 4194304
```

## Флаги

- `-hidden` — включать скрытые файлы и директории.
- `-max-file-size` — максимальный размер файла в байтах. `0` отключает ограничение.
- `-output` — записать отчет в файл.
- `-copy` — скопировать отчет сразу в буфер обмена macOS.
- `-extensions` — фильтр по расширениям: `go,md,yaml`.
- `-summary-only` — вывести только статистику.
- `-no-tree` — не печатать дерево директорий.
- `-no-contents` — не печатать содержимое файлов.
